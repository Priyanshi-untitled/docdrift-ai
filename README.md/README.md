## рҹ§  What Gets Detected

### Documentation Files (`.md`, `.txt`, `.rst`)
- вқҢ Empty or near-empty content (< 20 words)
- вқҢ Missing critical README sections (Install / Usage / API / License)
- вқҢ TODO / FIXME markers inside docs
- вқҢ Vague placeholder language (`stuff`, `things`, `foo`, `bar`)
- вқҢ Misleading hype claims (`"best"`, `"fastest"`, `"zero config"`, `"100%"`)

### Source Code (`.js`, `.ts`, `.py`, `.go`, etc.)
- вқҢ TODO / FIXME / HACK markers left in production
- вқҢ Excessive `console.log` debug statements
- вқҢ Large files (100+ lines) with < 3% comment coverage
- вқҢ Empty comment blocks

### Config Files (`package.json`, `.yaml`, `.toml`)
- вқҢ Hardcoded secrets / API keys
- вқҢ Missing `description` or `version` in `package.json`
- вқҢ `main` field pointing to a non-existent file
- вқҢ Malformed JSON
